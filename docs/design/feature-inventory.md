# 功能清单

## 当前版本（0.1.0）

> 当前处于项目初始化阶段，以下为设计阶段的功能清单。实现状态将随着开发进展更新。

### 类型系统

| 功能 | 状态 | 说明 |
|---|---|---|
| 基础类型 | 📋 设计中 | Int、Nat、Float、Bool、String、Bytes、Char、Regex、Duration、Unit、Path |
| 和类型 | 📋 设计中 | Maybe、Result、自定义和类型 |
| 代数数据类型 | 📋 设计中 | 乘积类型与和类型的组合 |
| 模式匹配 | 📋 设计中 | 和类型、列表、映射、守卫子句 |
| 类型推断 | 📋 设计中 | 自动推断局部变量和函数返回类型 |
| 泛型 | 📋 设计中 | 参数化多态 |
| 参数验证器 | 📋 设计中 | range、length、regex、enum、custom，链式组合 |

### 命令系统

| 功能 | 状态 | 说明 |
|---|---|---|
| 命令函数抽象 | 📋 设计中 | 将 Linux 命令抽象为带签名的函数 |
| CDF 命令描述文件 | 📋 设计中 | 定义命令的精确签名 |
| 内置签名库 | 📋 设计中 | 核心命令的预置签名 |
| 签名自动推断 | 📋 设计中 | 优先通过 man 手册获取帮助信息，回退到 --help/-h；支持子命令识别与独立签名建立 |
| 项目级自定义签名 | 📋 设计中 | 项目目录中提供自定义签名 |

### 运行时

| 功能 | 状态 | 说明 |
|---|---|---|
| dlopen 命令加载 | 📋 设计中 | 直接加载命令二进制的入口函数 |
| 结构化参数传递 | 📋 设计中 | 以结构化数据而非 argv 传递参数 |
| ptrace 透明适配层 | 📋 设计中 | 适配未标准化的命令 |
| fork/exec 回退 | 📋 设计中 | 传统执行模式的回退方案 |

### 管道与组合

| 功能 | 状态 | 说明 |
|---|---|---|
| 管道操作符 | 📋 设计中 | 值作为最后一个参数传入 |
| 惰性求值 | 📋 设计中 | 默认惰性求值策略 |
| 高阶函数 | 📋 设计中 | map、filter、fold、reduce 等 |

### 安全

| 功能 | 状态 | 说明 |
|---|---|---|
| 最小权限原则 | 📋 设计中 | 默认工作目录访问，显式权限声明 |
| 能力安全 | 📋 设计中 | 能力获取、使用、丢弃机制 |
| Namespace 沙箱 | 📋 设计中 | mount namespace + PID namespace 隔离 |
| 容器环境检测 | 📋 设计中 | 避免嵌套命名空间 |
| 权限异常详细报告 | 📋 设计中 | 结构化异常，含资源类型、路径、所需能力、源码位置、拒绝原因、修改建议 |
| 单命令权限注解 | 📋 设计中 | 通过 with capabilities 为单个命令指定精确权限集合 |
| 作用域权限声明 | 📋 设计中 | 通过 with capability 作用域语法临时授予能力 |
| CDF 驱动的行为验证 | 📋 设计中 | 输出类型不匹配、隐式 IO/网络访问检测 |
| 二进制完整性校验 | 📋 设计中 | SHA-256 哈希校验命令二进制未被篡改 |
| CDF 密码学签名 | 📋 设计中 | Ed25519 签名确保签名来源可信 |
| Seccomp 系统调用过滤 | 📋 设计中 | 基于 CDF 自动推导系统调用过滤规则 |
| 单命令沙箱隔离 | 📋 设计中 | 高风险命令在独立 namespace 中执行 |
| 信任分级策略 | 📋 设计中 | trusted / verified / sandboxed / denied 四级 |

### IO 与数据

| 功能 | 状态 | 说明 |
|---|---|---|
| Stream 类型 | 📋 设计中 | 惰性流，mmap/分块/非阻塞 IO |
| Maybe / Result | 📋 设计中 | 显式错误处理，? 操作符 |
| ? 操作符 | 📋 设计中 | 自动解包 Ok，传播 Err |
| Effect 类型 | 📋 设计中 | 结构化 IO 操作管理 |

### 语法与工具

| 功能 | 状态 | 说明 |
|---|---|---|
| 表达式导向语法 | 📋 设计中 | 借鉴 Elm/Haskell/Rust |
| REPL 交互环境 | 📋 设计中 | 结构化 REPL，语法高亮，错误报告 |
| 模块系统 | 📋 设计中 | 导入导出，默认私有，限定导入 |
| 解构赋值 | 📋 设计中 | 与模式匹配结合 |
