# 讨论记录：语法演进与设计决策

## 讨论背景

语法设计从初始草案逐步演进，经历了多轮讨论和调整。本文记录所有语法变更的讨论过程、替代方案和最终决策。

## 参与方

- AI Agent（方案分析）
- 项目维护者（决策裁定）

## 讨论内容

### 议题 1：无参函数（`() -> T`）是否支持

| 方案 | 要点 |
|------|------|
| 支持 `() -> T` | 回调/工厂函数/thunk 场景需要；与主流函数式语言一致 |
| 不支持 | `IO T` 已是延迟值；默认惰性求值；`()` 参数无意义 |

**最终决策**：**不支持**。替代方案：
- 回调/工厂函数：`IO T` 代替 `() -> IO T`，每次 `<-` 即执行
- 延迟计算：纯函数靠惰性绑定，无需 thunk 包装
- `Unit -> T` 不做替代——函数取 Unit 参数无意义

### 议题 2：字符串插值语法

| 方案 | 要点 |
|------|------|
| 不引入插值 | `++` 拼接足够；Elm 无插值 |
| `{expr}` 内嵌（JS/模板风格） | 简洁但需区分 `{` 字面量 |
| `f"..."` 前缀（Python 风格） | 显式区分插值与普通字符串；格式说明对齐 Python |

**最终决策**：**`f"..."` 前缀 + `{expr}` 嵌入 + `:` 格式说明**。格式说明与 Python 3 Format Specification Mini-Language 保持一致。

### 议题 3：前缀字面量引用符号

| 阶段 | Path | Regex | f-string | 问题 |
|------|------|-------|----------|------|
| 初始 | `path"..."` | `` regex`...` `` | `f"..."` | 引用符号不统一 |
| 第 1 次调整 | `` p`...` `` | `` r`...` `` | `` f`...` `` | 反引号与 markdown 冲突，解析器复杂度高 |
| 第 2 次调整 | `p"..."` | `r"..."` | `f"..."` | 双引号统一，前缀字面量为原始字符串，仅 `"` 需转义 |

**最终决策**：**统一双引号**。前缀字面量为原始字符串，不处理转义序列，仅 `"` 需 `\"` 转义。

### 议题 4：泛型语法

| 方案 | 示例 | 问题 |
|------|------|------|
| 尖括号 | `List<Int>` | Haskell 不兼容；`<` 与比较运算符冲突 |
| 空格分隔（Elm 风格） | `List Int` | 与函数应用区分需上下文；嵌套需括号 |

**最终决策**：**空格分隔 Elm 风格**。嵌套用括号分组 `IO (Result FileType IOError)`。

### 议题 5：函数类型与调用

| 维度 | 初始设计 | 最终决策 |
|------|---------|---------|
| 多参数类型 | `(Int, Int) -> Int` | `Int -> Int -> Int`（柯里化） |
| 参数分隔 | `f(x, y)` 逗号 | `f x y` 空格 |
| 无参函数 | `() -> T` 支持 | 不支持 |
| 别名 | `type alias` 关键字 | 仅函数类型 `type LongFunc = ...` |

### 议题 6：`let` 绑定语法

| 方案 | 单绑定 | 多绑定 |
|------|--------|--------|
| 统一用 `let` | `let x = 5` | `let x = 5; y = 6 in x + y` |
| 单条无 `let` | `x = 5` | `let x = 5; y = 6 in x + y` |

**最终决策**：单条无 `let`；多条 `let ... in`。

### 议题 7：List 模式匹配

| 方案 | 示例 |
|------|------|
| Cons 风格 | `x :: xs`、`x :: y :: rest` |
| 数组风格 | `[x, *xs]`、`[x, y, *rest]` |

**最终决策**：**数组风格** `[x, *xs]`。更直观，与 List 字面量形式一致。

### 议题 8：Map 字面量语法

| 方案 | 示例 |
|------|------|
| 胖箭头 | `#{ "a" => 1 }` |
| 等号 | `#{ "a" = 1 }` |

**最终决策**：**等号** `#{ "a" = 1 }`。与 Record 字面量风格一致。

### 议题 9：点调用语义

| 方案 | 允许 | 禁止 |
|------|------|------|
| 通用点调用 | 字段访问 + 方法调用 | — |
| 受限点调用 | 仅字段投影和元组索引 | 函数调用 |

**最终决策**：**受限点调用**。函数通过模块导入调用，`.` 仅用于 `record.name` 和 `tuple.0`。

### 议题 10：模块导入/导出

| 方案 | 导入语法 | 导出语法 |
|------|---------|---------|
| 初始 | `import List (map)` | `pub` 关键字 |
| v1（已弃用） | `import List as L with (map as m)`（`as` `with` 可组合） | `module List export (map)` 无 `pub` |
| 最终（v2） | `import List as L` 或 `import List with (map as m)`（互斥，不可组合） | `module List export (map)` 无 `pub` |

### 议题 11：`?` / `=?` 操作符位置

| 方案 | 语法 | 说明 |
|------|------|------|
| 表达式后 | `(expr)?` | 优先级低，需括号包裹 |
| 函数名后 | `funcName? args` | 作用于函数返回值 |
| IO 绑定时 | `name <-? expr` | 作用于绑定表达式的 Result |
| 纯绑定时 | `name =? expr` | 作用于纯 Result 表达式 |
| Stream 上 | `result?` 逐元素解包 | REJECTED——用 `filterMap toMaybe` |

**最终决策**：**纯绑定 `=?` + IO 绑定 `<-?`**。函数名后缀 `funcName?` 被移除，早返回仅限变量绑定时发生。Stream 上不支持逐元素解包。

### 议题 12：`<-` 解包语义

`<-` 触发 IO 求值，剥离 `IO` 包装，将内部值绑定到名字。无 `<-` 时操作仍被执行但结果丢弃。

| 写法 | 含义 |
|------|------|
| `expr` | 执行（类型 `IO T`），丢弃 `T` |
| `name <- expr` | 执行，解包 `T` 绑定到 `name` |
| `name <-? expr` | 执行，解包 IO + Result，Err 早返回 |

### 议题 13：脚本入口

| 方案 | 规则 |
|------|------|
| A. 强制 `main` | 必须 `main : IO Unit` |
| B. 顶层即入口 | 顶层 IO 按顺序执行 |
| C. 混合 | 有 `main` 从 `main` 启动；无 `main` 执行顶层 IO；库文件不执行 |

**最终决策**：**方案 C**。
- `main : IO Unit`（无参数）或 `main : List String -> IO Unit`（有参数）
- 无 `main` 时有顶层 IO 表达式则顺序执行
- 有 `module export` 的库文件不自动执行

### 议题 14：Stream 设计

| 维度 | 初始设计 | 最终决策 |
|------|---------|---------|
| 构造语法 | `stream expr` 关键字 | 已移除。`Stream.fromList`/`Stream.range`/`Stream.readLines` |
| IO Stream 消费 | 直接链式 | 必须 `<-` 解包后消费 |
| 构造错误处理 | 未定义 | `IO (Result (Stream String) IOError)`，`<-?` 解包 |
| 运行时错误处理 | 未定义 | 静默终止流；安全版本 `readLinesSafe` 每行包裹 `Result` |
| 元素类型 | 纯值 `t` | 不变。运行时错误不暴露为元素级 `Err` |

### 议题 15：注释语法

| 方案 | 示例 |
|------|------|
| `--` 风格 | `-- 注释` |
| `//` 风格 + doc comment | `// 注释` |

**最终决策**：**`//` 风格**。类型/函数/模块上的 `//` 为文档注释，支持 Markdown。

### 议题 16：文件后缀

**最终决策**：**`.kun`**。

## 一致性决议（历史变更对照）

以下为语法演进过程中解决的每一项不一致：

| 原不一致 | 决议 |
|---------|------|
| lambda 参数形式 | `\x ->` 单参数；`\x y ->` 多参数；`\(x, y) ->` 元组解构；`\{x, y} ->` Record 解构；`\[x, y] ->` List 解构 |
| 注释风格 | 使用 `//`，类型/函数/模块上的 `//` 为文档注释，支持 Markdown |
| 字面量引用风格 | `p"..."`（Path）、`r"..."`（Regex）、`f"..."`（插值字符串）统一使用双引号；前缀字面量为原始字符串 |
| 多行字符串 | `"""` 包裹（插值用 `f"""`），自动去公共缩进 |
| 字符串插值 | `f"..."` 前缀 + 大括号嵌入 + `:` 格式说明，编译期展开为 `toString`/格式化链 |
| 泛型语法 | Elm 风格空格分隔（`List Int`），不使用尖括号，嵌套用括号分组 |
| 函数类型 | 柯里化 `Int -> Int -> Int`，除非参数本身为元组 |
| 函数应用 | 空格分隔参数，无逗号；元组参数用圆括号包裹 |
| 名字绑定 | 均以 `a = b` 形式；`let ... in` 用于确保多条语句的唯一返回值 |
| List 模式 | `[a, *rest]` 替代 `a :: rest` |
| Map 字面量 | `#{ "a" = 1 }`（`=` 替代 `=>`）；Map 索引 `data["key"]`；Map 更新使用 `update` 语法 |
| `capability` -> `with caps` | `with capability`/`with capabilities` 统一为 `with caps` |
| 点调用语义 | 仅限积类型字段投影和元组索引，无函数调用 |
| `Stream` 构造 | `Stream.fromList`、`Stream.range` 替代 `stream` 关键字 |
| 模块导入 | `import List as L`（模块别名）或 `import List with (map as m)`（精选导入）互斥；`Maybe(*)` 变体导入语法 |
| 模块导出 | `module List export (map)` 声明语法，无 `pub` 关键字 |
| 导出语法 | 仅 `module export`，无 `pub` 关键字 |
| 类型别名 | 仅函数类型支持 `type LongFunc = ...`，非函数类型别名在导入时指定 |
| 三元表达式 | `condition ? expr1 : expr2` 简化 `if then else` |
| 函数组合 | 从左向右 `>>`, 从右向左 `<<` |
| 反向管道 | 减少括号嵌套 |
| 无参函数 | 不支持 `() -> T`。`IO T` 已是延迟值，`<-` 即执行；纯函数惰性求值靠绑定 |
| `=` / `=?` / `<-?` | `name = expr` 纯绑定；`name =? expr` 解纯 Result，Err 早返回；`name <-? expr` 解 IO + Result，Err 早返回 |
| `?` / `=?` 位置 | 纯绑定 `name =?` 和 IO 绑定 `name <-?`；函数名后缀 `funcName?` 已移除；Stream 上不支持逐元素解包，用 `filterMap toMaybe` |
| `stream` 关键字 | 已移除。纯构造用 `Stream.fromList`/`Stream.range`，IO 构造用 `Stream.readLines` |
| IO Stream 消费 | IO Stream 必须通过 `<-` 解包后才能进行 `map`/`filter`/`iter` 等消费操作 |
