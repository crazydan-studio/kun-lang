# 讨论记录：命令函数设计——结构化输出与安全约束

## 讨论背景

命令函数是 Kun 将 Linux 命令抽象为类型安全函数的核心机制。但最初的 CDF 设计对输出格式的处理方式存在争议——是否应在 CDF 中声明 `--json`、`-l` 等格式相关参数，以及命令函数的返回值是否应该结构化。

## 参与方

- AI Agent（方案分析与实施）
- 项目维护者（决策裁定）

## 讨论内容

### 议题 1：命令输出是否需要结构化？

| 立场 | 理由 |
|------|------|
| 不需要，字符串输出足够 | `exec` 作为逃生口，用户在 Kun 层自行解析 |
| 需要结构化 | 否则命令函数相比 `exec` 只多了参数校验，价值不足以支撑 CDF 体系复杂度 |

**结论**：命令函数必须返回结构化类型。调用 `ls` 应获得 `Stream { name : Path, size : Int, mtime : DateTime }`，而非 `Stream String`。

### 议题 2：CDF 是否应包含输出格式相关参数？

| 方案 | 说明 |
|------|------|
| 包含 `--json`、`-l`、`-h` 等 | 用户控制输出格式，CDF 声明 `when json -> / when !json ->` |
| 不包含——运行时自动处理 | CDF 只声明影响结果集合的参数，运行时会自动选择最佳输出格式（如自动加 `--json`） |

**结论**：CDF **不**包含输出格式相关参数。`-l`、`-h`、`--json`、`--format` 等不映射到命令函数的参数，运行时自动选择最佳输出格式用于结构化解析。

### 议题 3：命令函数参数形式

| 方案 | 说明 |
|------|------|
| 散列 flag + 位置参数 | `ls : { all : Bool } -> Maybe Path -> Maybe Path -> IO ...` |
| 统一 Record 参数 | `ls : { all : Bool, path0 : Maybe Path, ... } -> IO ...` |

**结论**：统一 Record 参数。所有 `flag`/`option`/`positional` 合并到同一 Record 类型中。

### 议题 4：`exec` 原语是否保留？

| 方案 | 说明 |
|------|------|
| 保留 `exec` 作为逃生口 | 无 CDF 的命令通过 `exec` 调用 |
| 移除 `exec` | 无 CDF 则命令不可用，统一走命令函数路径 |

**结论**：**移除 `exec`**。命令调用统一走 CDF 路径，消除安全逃逸。

### 议题 5：`process.exec` 能力是否必要？

| 方案 | 说明 |
|------|------|
| 保留，但自动推断直接调用命令 | 脚本直接调用的命令自动获得 `process.exec` 授权；模块引入的命令需显式声明 |
| 完全移除 | CDF 存在即授权 |

**结论**：完全移除 `process.exec`。命令函数通过 CDF 存在获得授权，CDF 即授权。

### 议题 6：`sudo`/`su` 如何映射？

| 方案 | 说明 |
|------|------|
| CDF 映射为命令函数 | `sudo` 本质是命令包装器，CDF 的参数模型无法表达"另一个命令调用" |
| 作为命令函数的隐式参数 `runAs` | 所有命令函数带有 `runAs : Maybe String`，通过 `process.run-as` 能力控制允许切换到的用户 |

**结论**：禁止 `sudo`/`su` 映射为命令函数。新增 `runAs` 隐式参数 + `process.run-as` 能力。

### 议题 7：文本变换类命令是否映射？

| 命令 | 是否映射 | 理由 |
|------|---------|------|
| `sed`、`awk`、`sort`、`uniq`、`cut`、`tr` | ❌ | Kun 标准库可直接替代 |
| `cat`、`wc`、`tee` | ❌ | `readFile`/`writeFile` 覆盖 |
| `grep` | ✅ | 跨文件搜索、大文件性能——Kun 无法替代 |
| `tar`、`gzip`、`rsync` | ✅ | 影响文件系统，需要能力系统集成 |
| `sudo` | ❌ | 由 `runAs` 替代 |

### 议题 8：内置签名库范围

所有影响文件系统或 OS 状态但 Kun 无法原生实现的命令都应映射，即使无结构化输出——CDF 映射提供能力系统集成（参数验证、沙箱路径隔离、seccomp 规则）。

## 结论

1. 命令函数返回结构化类型，运行时自动解析输出
2. CDF 不声明输出格式参数，运行时自动处理
3. 命令参数统一为 Record 类型
4. 移除 `exec` 原语，无 CDF 则命令不可用
5. 完全移除 `process.exec` 能力，CDF 存在即授权
6. 新增 `runAs` 隐式参数 + `process.run-as` 能力，禁止 `sudo`/`su` 映射
7. 文本变换类命令不映射，功能由标准库覆盖
8. 内置签名库按"能力系统集成价值"确定映射范围
