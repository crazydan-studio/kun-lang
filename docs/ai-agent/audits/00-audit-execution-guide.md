# 审计执行指南

## 审计类型

### 文档审计

挑战需求、设计、架构文档的质量和完整性。使用 `docs/ai-agent/skills/document-audit-prompt.md`。

### 计划审计

在实施前挑战执行计划的可行性。使用 `docs/ai-agent/skills/plan-audit-prompt.md`。

### 闭合审计

独立审查变更的完成度。使用 `docs/ai-agent/skills/closure-audit-prompt.md`。

### 多维审计

跨维度的风险挑战。使用 `docs/ai-agent/skills/multi-dimensional-audit-prompt.md`。

### 开放式审计

对抗性思维发现隐藏风险。使用 `docs/ai-agent/skills/open-ended-audit-prompt.md`。

## 执行要求

- 审计必须由独立的子代理/审查者执行
- 审计结论必须记录到本目录
- 发现的问题必须跟踪到解决
