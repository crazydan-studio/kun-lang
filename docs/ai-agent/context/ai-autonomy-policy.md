# AI 自治策略

## 自治级别定义

| 级别 | 含义 | 适用场景 |
|---|---|---|
| `implement` | AI 可在阅读需求 + owner doc + 验证命令后自主实施 | 文档编写、配置修改、模板文件创建 |
| `plan-first` | AI 可起草计划，但实施需等待计划审计通过 | 新模块开发、跨模块重构 |
| `ask-first` | AI 必须在代码/行为变更前询问 | 类型系统核心变更、安全模型变更 |
| `research-only` | 仅检查/总结，不做行为变更 | 架构调研、技术方案评估 |
| `blocked` | 必须等待阻塞项解决 | 存在未解决的技术风险 |

## 当前自治级别

**总体级别**：`implement`

## 受保护区域

| 区域 | 规则 | 必要证据 |
|---|---|---|
| 类型系统核心 | 需要 `plan-first` 或以上 | 更新后的类型系统设计文档 |
| 运行时安全模型 | 需要 `ask-first` | 人工明确批准 |
| Cmd 命令系统 | 需要 `plan-first` | 命令系统设计文档与测试方案 |
| Landlock/seccomp 安全子系统 | 需要 `ask-first` | 安全评估报告 |
| 许可证文件 | 需要 `ask-first` | 人工明确批准 |

## 自治级别变更

自治级别的变更由项目维护者通过修改本文件来执行。AI 不应自行变更自治级别。
