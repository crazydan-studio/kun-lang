# 项目愿景

## 核心问题

Shell 脚本是 Linux 系统管理自动化的重要工具，但存在以下根本性问题：

1. **缺乏类型安全**：变量无类型约束，参数传递全凭字符串拼接，运行时才能发现错误
2. **错误处理薄弱**：错误码与异常信号混杂，缺乏结构化的错误处理机制
3. **数据结构原始**：只有字符串和数组，无法表达复杂的数据关系
4. **组合能力有限**：管道仅传递文本流，无法传递结构化数据
5. **安全风险高**：脚本拥有与用户相同的权限，缺乏细粒度的访问控制

## 愿景

设计一款函数式、强静态类型、代数数据类型、结构化 + 不可变数据、表达式导向语法、高性能的 Linux 脚本语言——Kun（鲲），从根本上消除 Shell 脚本所存在的各种问题，同时保持 Unix 哲学中"小程序组合完成复杂任务"的核心理念。

## 核心理念

### 命令即函数

将每一个 Linux 命令抽象为一个具有确定签名的函数值：参数类型定义了该命令接受的所有参数及其类型；输出类型定义了该命令的标准输出结构；错误类型定义了该命令可能返回的所有错误。确保命令的组合不再是字符串拼接，而是类型驱动的函数复合。

### 类型驱动组合

通过 `.cmd.kun` 文件扩展或自定义命令签名。为常见的核心命令提供内建 Primitive 实现，对于其他命令，通过 `kun cmd init` 从 `man`/`--help` 自动生成 `.cmd.kun` 骨架。

### 安全即默认

采用最小权限原则和能力安全。脚本默认只能访问其工作目录及其子目录，所有超出此范围的资源访问都必须通过显式的权限声明来获得。基于 Linux namespace 的轻量级沙箱能力，支持在隔离的 mount namespace 和 PID namespace 中执行脚本。

### 表达式导向

语法设计借鉴 Elm、Haskell 和 Rust（以 Elm 语法结构为主），确保简洁、统一、一致。所有数据均必须赋初值，消除 null，支持自动类型推断。

## 实现策略

采用 Zig 作为宿主语言构建一个单体的、无库依赖的、轻量级的二进制脚本执行器和结构化的 REPL 交互环境。

运行时使用 Linux 的 `dlopen`/`dlsym` 接口定位命令二进制的 main 函数并直接调用，以结构化数据而非 argv 数组的方式传递参数，实现高性能的命令执行。
