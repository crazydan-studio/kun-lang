# 模块边界

## 模块划分

```
kun-lang/
├── 解释器核心
│   ├── 词法分析器（Lexer）
│   ├── 语法分析器（Parser）
│   ├── 类型检查器（Type Checker）
│   └── AST（抽象语法树）
├── 运行时（Runtime）
│   ├── 命令加载器（dlopen/dlsym）
│   ├── 透明适配层（ptrace stub）
│   ├── 结构化参数序列化
│   └── 结果反序列化
├── 命令签名系统
│   ├── CDF 解析器
│   ├── 内置签名库
│   ├── 签名自动推断（优先 man 手册，回退 --help/-h）
│   │   └── 子命令识别与独立签名建立
│   └── 参数验证器
├── 安全子系统
│   ├── 权限声明解析
│   │   ├── 脚本级权限解析
│   │   └── 作用域权限解析（with caps）
│   ├── 能力管理器
│   ├── Namespace 沙箱
│   ├── 容器环境检测
│   ├── 命令级安全
│   │   ├── CDF 行为契约验证
│   │   ├── 二进制完整性校验（SHA-256）
│   │   ├── CDF 密码学签名验证（Ed25519）
│   │   ├── Seccomp 过滤规则生成（基于 CDF）
│   │   ├── 单命令沙箱隔离
│   │   └── 信任分级策略
├── 标准库
│   ├── 数据类型（List、Map、Set、Stream 等）
│   ├── 管道与高阶函数
│   ├── 模式匹配
│   └── IO 操作
└── REPL
    ├── 交互式环境
    ├── 语法高亮
    └── 错误报告
```

## 模块职责

### 解释器核心

负责将 Kun 源代码解析为 AST 并进行类型检查。包含词法分析、语法分析、类型推断和类型检查。这是编译器的前端部分，不涉及任何 IO 或系统调用。

### 运行时

负责执行编译后的代码。通过 `dlopen`/`dlsym` 加载命令二进制文件，以结构化数据方式传递参数。对于未适配的命令，提供 ptrace 拦截适配层。无 CDF（Command Description File，命令描述文件）的命令不可调用。

### 命令签名系统

负责管理和维护命令的签名信息。CDF 文件定义精确签名，内置签名库提供核心命令的签名，自动推断模块根据帮助信息生成近似签名。自动推断优先通过 man 手册获取命令文档（最详尽、最精确），当 man 手册不可用时回退到 `--help`/`-h` 输出。解释器能够识别命令的子命令结构（如 `git commit`、`docker run`），并为每个子命令递归获取帮助信息、分别建立独立的签名。参数验证器在运行时对参数值进行额外验证。

### 安全子系统

实现多层安全模型。解析二级权限声明（脚本级、表达式级），管理能力生命周期，基于 Linux namespace 创建沙箱环境，并检测容器化环境以避免嵌套命名空间。当权限检查失败时，生成结构化的 `PermissionError` 异常。在命令级安全层面，通过 CDF 行为契约验证命令的实际行为是否与签名一致，通过 SHA-256 哈希校验命令二进制完整性，通过 Ed25519 签名验证 CDF 来源可信性，基于 CDF 自动推导 seccomp-BPF 系统调用过滤规则，对高风险命令提供独立 namespace 沙箱隔离，并根据信任分级策略自动调整安全措施的严格程度。

### 标准库

提供内置的数据类型和函数。包括列表、映射、集合、流等数据结构，管道和高阶函数组合工具，模式匹配机制，以及结构化的 IO 操作管理。

### REPL

提供交互式的开发和调试环境，支持语法高亮、自动补全和结构化的错误报告。

## 模块依赖关系

```
REPL → 解释器核心 → 运行时
                        ↓
解释器核心 ← 命令签名系统
                        ↓
            安全子系统 ← 运行时
                        ↓
                   标准库 ← 运行时
```

解释器核心依赖命令签名系统进行类型检查；运行时依赖安全子系统进行权限控制和沙箱管理；标准库由运行时加载并提供给用户代码使用；REPL 是解释器核心的交互式包装。
