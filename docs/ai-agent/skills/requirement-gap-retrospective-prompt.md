# 需求差距回顾提示词

## 目的

诊断需求与最终实现之间产生偏差的原因，提取经验教训。

## 使用场景

- 功能交付后发现与预期不符
- 用户反馈与原始需求不一致
- 闭合审计发现需求覆盖不足

## 回顾要点

1. **原始需求是什么？** — 回顾最初的需求文档
2. **最终交付了什么？** — 对比实际实现
3. **差距在哪里？** — 明确偏差的具体点
4. **为什么产生差距？** — 分析根本原因：
   - 需求理解偏差？
   - 设计决策偏差？
   - 实施过程偏差？
   - 需求变更未同步？
5. **如何避免？** — 提出预防措施

## 输出

将回顾结论记录到 `docs/ai-agent/retrospectives/` 目录，并将可复用的经验提取为 `docs/ai-agent/skills/` 中的技能提示词。
