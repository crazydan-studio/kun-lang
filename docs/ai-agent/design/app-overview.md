# 应用概览

## Kun 语言概览

Kun（鲲）是一款面向 Linux 的函数式脚本语言，其核心目标是消除传统 Shell 脚本的种种问题，同时保留 Unix 哲学中"小程序组合完成复杂任务"的精髓。

## 类型系统

### 基础类型

| 类型 | 说明 |
|---|---|
| `Int` | 整数 |
| `Nat` | 自然数（非负整数），独立于 `Int` |
| `Float` | 浮点数 |
| `Bool` | 布尔值 |
| `String` | 文本字符串 |
| `Bytes` | 二进制数据，与 `String` 明确区分 |
| `Char` | 单个字符 |
| `Regex` | 正则表达式 |
| `Duration` | 时间段 |
| `Unit` | 单元类型，表示无返回值 |
| `Path` | 文件系统路径 |

### 复合类型

| 类型 | 说明 |
|---|---|
| `List` | 顺序序列，支持索引访问和模式匹配解构（Rope 实现） |
| `Map` | 映射表，提供对数复杂度的查找、插入和删除 |
| `Set` | 集合，元素唯一且无序 |
| `Stream` | 惰性流，支持 mmap 或分块读取文件、非阻塞 IO |
| `Tuple` | 元组 |

### 和类型（Sum Types）

| 类型 | 变体 | 说明 |
|---|---|---|
| `Maybe` | `Just a` / `Nothing` | 表示值可能存在或不存在 |
| `Result` | `Ok a` / `Err e` | 表示操作可能成功或失败 |

`?` 操作符可在返回 `Result` 类型的函数中自动解包 `Ok` 变体并传播 `Err` 变体，实现早返回。

### 标准库补充类型

脚本领域特定的类型（`Port`、`Pid`、`Signal`、`ExitCode`、`DateTime`、`User`/`Group`、`IpAddress`、`Errno`、`FileType`、`IOError` 等）由[标准库](standard-library.md)以 ADT 或 newtype 形式定义，详见独立文档。

## 命令函数系统

每个 Linux 命令被抽象为具有确定签名的函数值：

- **参数类型**：定义命令接受的所有参数（标志位、位置参数、命名参数）及其类型
- **输出类型**：定义命令的标准输出结构（行列表、JSON 结构、原始字节等）
- **错误类型**：定义命令可能返回的所有错误变体

### 命令描述文件（CDF）

CDF 定义命令的精确签名。常见核心命令（ls、cat、grep、find、sed、awk 等）内置了精确签名。其他命令通过 man 手册和 `--help`/`-h` 自动推断，并支持项目级自定义。

### 签名自动推断

对于未内置签名且无 CDF 文件的命令，解释器按照以下优先级自动获取帮助信息并推断签名：

1. **man 手册（首选）**：优先通过 `man` 命令获取命令的完整文档。man 手册通常包含最详尽的参数说明、选项列表和用法示例，能够提供比 `--help` 更精确和全面的签名信息。解释器解析 man 页面中的 OPTIONS、ARGUMENTS、DESCRIPTION 等段落，提取参数名称、类型约束和描述信息
2. **`--help` / `-h`（回退）**：当 man 手册不可用（如未安装对应 man page）时，回退到 `--help` 或 `-h` 标志获取帮助信息
3. **AI 辅助整理**：当自动推断结果不够精确时，解释器可输出合适的提示词，交由 AI（如 LLM）整理成标准 CDF 文件

#### 子命令识别与签名推断

许多现代 CLI 工具采用子命令结构（如 `git commit`、`docker run`、`kubectl get pods`）。Kun 的签名推断系统能够识别和区分命令的子命令，并为每个子命令分别建立独立的签名：

- **子命令发现**：通过 man 手册的 COMMANDS 或 SUBCOMMANDS 段落，或 `--help` 输出中的 `Usage: <cmd> <subcommand> ...` 模式，自动发现命令支持的子命令列表
- **子命令帮助获取**：对于识别到的每个子命令，递归地获取其帮助信息。优先尝试 `man <cmd>-<subcommand>`（如 `git-commit`）或 `man <cmd> <subcommand>`，回退到 `<cmd> <subcommand> --help`
- **独立签名映射**：每个子命令作为独立的函数值，拥有自己的参数类型和输出类型。主命令的签名则映射为子命令名称到子命令签名的映射类型
- **嵌套子命令支持**：支持多层嵌套的子命令结构（如 `kubectl get pods -o json`），递归解析直到叶节点命令

### 参数验证器

每个参数可附加验证器表达式，在运行时验证参数值。内置验证器包括：

- `range`：数值范围检查
- `length`：字符串长度检查
- `regex`：格式检查
- `enum`：枚举值检查
- `custom`：自定义验证函数

验证器支持链式组合。

## 管道与组合

内置管道机制和高阶函数实现命令组合：

- **管道操作符**：将左侧表达式的值作为右侧函数的最后一个参数传入
- **惰性求值**：默认采用惰性求值策略
- **高阶函数**：map、filter、fold、reduce 等

## 模式匹配

支持多种模式匹配形式：

- **和类型模式**：匹配 `Maybe`、`Result` 等和类型的变体
- **列表模式**：匹配列表结构（空列表、`[a, *rest]`、特定元素序列）
- **映射模式**：匹配特定键的存在
- **守卫子句**：附加到模式分支的额外布尔条件

## 错误处理与诊断

### 权限异常报告

当脚本尝试访问未被授权的资源时，Kun 抛出结构化的 `PermissionError`，包含：

- 被拒绝的资源类型和路径
- 所需能力的名称
- 源码位置（文件名、行号、列号）
- 权限被拒绝的具体原因
- 可直接复制粘贴的权限声明语法模板（修改建议）

详细的异常设计请参见 [安全角色与权限模型](roles-and-permissions.md#权限异常报告)。

## 安全模型

### 最小权限原则

脚本默认只能访问其工作目录及其子目录。所有超出此范围的资源访问（网络访问、环境变量、文件系统、进程管理等）必须通过显式权限声明获得。权限支持三级粒度：脚本级声明（全局基线）、作用域级声明（临时扩缩）、单命令级注解（精确约束）。

### 单命令权限与作用域权限

通过 `with capabilities` 为单个命令指定精确的权限集合，通过 `with capability` 作用域语法临时授予能力。详细设计请参见 [安全角色与权限模型](roles-and-permissions.md#单命令权限分配)。

### 能力安全

权限获取的三种途径：
1. 运行时在启动时根据权限声明授予
2. 由父脚本显式传递
3. 由运行时在用户确认后动态授予

能力一旦获取，只能被使用或丢弃，不能被复制或伪造。

### Namespace 沙箱

基于 Linux namespace 的轻量级沙箱，支持 mount namespace 和 PID namespace 隔离。在容器化环境（Docker、Kubernetes）中自动检测并避免嵌套命名空间。

## 供应链安全

Kun 通过 CDF 行为契约、二进制完整性校验、密码学签名、seccomp 系统调用过滤、单命令沙箱隔离和信任分级策略构成多层供应链防御体系。详细设计请参见 [供应链安全](supply-chain-security.md)。

## 与容器化方案的对比

Kun 的安全模型在隔离粒度、启动开销、组合方式上与 Docker/gVisor/Firecracker 等方案有根本差异，二者为互补关系。详细对比请参见 [安全角色与权限模型](roles-and-permissions.md#与容器化方案的对比)。

## 语法设计

语法借鉴 Elm、Haskell 和 Rust（以 Elm 为主），深度融合 Unix 哲学，确保简洁、统一、一致。所有数据必须赋初值，消除 null，支持自动类型推断。

## 运行时执行

### 主要机制：dlopen/dlsym

通过 dlopen 直接加载命令二进制文件中的入口函数，在当前进程地址空间内执行命令逻辑，以结构化数据而非 argv 数组传递参数。

### 标准化入口接口

定义 C ABI 兼容的命令接口规范。命令开发者通过链接运行时库提供标准化入口。

### 透明适配层

对于未适配的现有命令：
1. 通过 ptrace 拦截命令的 execve 并注入适配 stub
2. 在 stub 中重新导出命令的 main 函数为标准化入口
3. 回退到传统的 fork/exec 模型

## 函数与模块

- 支持导入导出函数，提供逻辑复用
- 默认私有，限定导入，别名导入
- 结构化的 REPL 交互环境
