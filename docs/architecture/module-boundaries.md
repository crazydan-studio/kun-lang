# 模块边界

## 模块划分

```
kun-shell/
├── 解释器核心
│   ├── 词法分析器（Lexer）
│   ├── 语法分析器（Parser）
│   ├── 类型检查器（Type Checker）
│   └── AST（抽象语法树）
├── 运行时（Runtime）
│   ├── 命令加载器（dlopen/dlsym）
│   ├── 透明适配层（ptrace stub）
│   ├── 结构化参数序列化
│   └── 结果反序列化
├── 命令签名系统
│   ├── CDF 解析器
│   ├── 内置签名库
│   ├── 签名自动推断（优先 man 手册，回退 --help/-h）
│   │   └── 子命令识别与独立签名建立
│   └── 参数验证器
├── 安全子系统
│   ├── 权限声明解析
│   ├── 能力管理器
│   ├── Namespace 沙箱
│   └── 容器环境检测
├── 标准库
│   ├── 数据类型（List、Map、Array、Stream 等）
│   ├── 管道与高阶函数
│   ├── 模式匹配
│   └── IO 操作
└── REPL
    ├── 交互式环境
    ├── 语法高亮
    └── 错误报告
```

## 模块职责

### 解释器核心

负责将 Kun 源代码解析为 AST 并进行类型检查。包含词法分析、语法分析、类型推断和类型检查。这是编译器的前端部分，不涉及任何 IO 或系统调用。

### 运行时

负责执行编译后的代码。通过 `dlopen`/`dlsym` 加载命令二进制文件，以结构化数据方式传递参数。对于未适配的命令，提供 ptrace 拦截适配层，最终回退到传统的 fork/exec 模型。

### 命令签名系统

负责管理和维护命令的签名信息。CDF 文件定义精确签名，内置签名库提供核心命令的签名，自动推断模块根据帮助信息生成近似签名。自动推断优先通过 man 手册获取命令文档（最详尽、最精确），当 man 手册不可用时回退到 `--help`/`-h` 输出。解释器能够识别命令的子命令结构（如 `git commit`、`docker run`），并为每个子命令递归获取帮助信息、分别建立独立的签名。参数验证器在运行时对参数值进行额外验证。

### 安全子系统

实现最小权限原则和能力安全模型。解析权限声明，管理能力生命周期，基于 Linux namespace 创建沙箱环境，并检测容器化环境以避免嵌套命名空间。当权限检查失败时，生成结构化的 `PermissionError` 异常，包含被拒绝的资源类型与路径、所需能力名称、源码位置、拒绝原因和有针对性的修改建议（可复制粘贴的权限声明语法模板）。

### 标准库

提供内置的数据类型和函数。包括列表、映射、数组、流等数据结构，管道和高阶函数组合工具，模式匹配机制，以及结构化的 IO 操作管理。

### REPL

提供交互式的开发和调试环境，支持语法高亮、自动补全和结构化的错误报告。

## 模块依赖关系

```
REPL → 解释器核心 → 运行时
                        ↓
解释器核心 ← 命令签名系统
                        ↓
            安全子系统 ← 运行时
                        ↓
                   标准库 ← 运行时
```

解释器核心依赖命令签名系统进行类型检查；运行时依赖安全子系统进行权限控制和沙箱管理；标准库由运行时加载并提供给用户代码使用；REPL 是解释器核心的交互式包装。
