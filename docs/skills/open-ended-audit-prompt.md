# 开放式审计提示词

## 目的

以对抗性思维发现隐藏风险，不受预设维度的限制。

## 使用场景

- 关键决策前
- 重大变更前
- 定期的健康检查

## 审计方法

扮演一个挑剔的审查者，尝试从各种角度挑战当前的设计和实现：

1. **假设什么会出错？** — 列出可能失败的所有场景
2. **如果用户这样做会怎样？** — 想象非预期的使用方式
3. **边界条件是什么？** — 极端输入、并发、资源耗尽等
4. **依赖是否可靠？** — 外部依赖的可用性和稳定性
5. **有哪些被忽略的细节？** — 错误处理、资源清理、日志等

## 输出格式

按风险严重程度排序：
- 🔴 高风险：可能导致系统故障或数据丢失
- 🟡 中风险：可能导致功能异常或性能下降
- 🟢 低风险：改进建议
