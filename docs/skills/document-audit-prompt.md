# 文档审计提示词

## 目的

挑战和验证需求、设计、架构文档的质量和完整性。

## 使用场景

- 新需求文档编写完成后
- 设计文档更新后
- 架构文档变更后
- 版本发布前

## 审计维度

1. **完整性**：文档是否覆盖了所有必要的方面？
2. **一致性**：文档之间是否存在矛盾？
3. **可行性**：文档中描述的方案是否可行？
4. **清晰性**：文档是否足够清晰，能否被 AI Agent 正确理解？
5. **可验证性**：文档中的验收标准是否可验证？

## 执行步骤

1. 阅读目标文档
2. 阅读相关的 owner docs
3. 按上述维度逐项检查
4. 记录发现的问题
5. 提出改进建议
