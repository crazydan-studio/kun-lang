# 多维审计提示词

## 目的

跨维度进行风险挑战，发现单一维度审计中可能遗漏的风险。

## 使用场景

- 重大架构变更前
- 版本发布前
- 长期项目里程碑评审时

## 审计维度

1. **需求维度**：需求是否完整、一致、无歧义？
2. **设计维度**：设计是否满足需求？是否过度设计或不足？
3. **架构维度**：架构是否合理？是否有技术债务？
4. **安全维度**：安全模型是否完善？是否有漏洞？
5. **可维护性维度**：代码和文档是否易于维护？
6. **性能维度**：是否有性能风险？

## 执行步骤

1. 阅读所有相关的 owner docs
2. 按每个维度逐一审计
3. 关注维度间的交叉影响
4. 记录所有发现的风险
5. 按严重程度排序并提出处理建议
